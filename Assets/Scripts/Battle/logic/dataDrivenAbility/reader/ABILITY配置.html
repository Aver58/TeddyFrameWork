基础属性

Name

技能名字和描述

"Name" : "光击阵"

AbilityBehavior

描述技能如何生效，生效后如何执行，可以配置多条行为。用来区分主动技能和被动技能。主要用于可以手动释放的大招技能的配置。

"AbilityBehavior" : [

    "ABILITY_BEHAVIOR_NO_TARGET",

    "ABILITY_BEHAVIOR_LINE_AOE"

]
AbilityBehavior 描述
ABILITY_BEHAVIOR_PASSIVE 被动技能，AI不会选取，不能释放，自动生效
ABILITY_BEHAVIOR_NO_TARGET 无目标技能，不需要选取目标或者点就能释放
ABILITY_BEHAVIOR_UNIT_TARGET 单位目标技能，需要目标释放
ABILITY_BEHAVIOR_POINT 点目标技能，释放时需要指定一个位置
ABILITY_BEHAVIOR_CHANNELLED 持续施法技能，施法者被眩晕等情况技能会被打断, 需要配合AbilityChannelTime来使用
ABILITY_BEHAVIOR_RADIUS_AOE 圆形AOE技能，需要配合AbilityAoeRadius来使用
ABILITY_BEHAVIOR_LINE_AOE 线形AOE技能，需要配合AbilityAoeLine来使用
ABILITY_BEHAVIOR_SECTOR_AOE 扇形AOE技能，需要配合AbilityAoeSector来使用

AbilityType

"AbilityType" : "ABILITY_TYPE_ULTIMATE"
AbilityType 描述
ABILITY_TYPE_ATTACK 普通攻击
ABILITY_TYPE_BASIC 普通技能
ABILITY_TYPE_ULTIMATE 大招

AbilityBranch

"AbilityBranch" : "ABILITY_BRANCH_PHYSICAL"
AbilityBranch 描述
ABILITY_BRANCH_PHYSICAL 物理类型的技能，可被缴械等物理沉默导致技能不可释放
ABILITY_BRANCH_MAGICAL 法术类型的技能，可被沉默等法术沉默导致技能不可释放

AbilityUnitTargetTeam

技能可选取的目标队伍

"AbilityUnitTargetTeam": "UNIT_TARGET_TEAM_FRIENDLY"
AbilityUnitTargetTeam 描述
UNIT_TARGET_TEAM_ENEMY 敌人
UNIT_TARGET_TEAM_FRIENDLY 友军
UNIT_TARGET_TEAM_BOTH 全部
UNIT_TARGET_TEAM_NONE 无

AbilityUnitTargetType

技能可选取的目标类型（目前这个字段没起作用，但是需要配置，要不技能会解析失败）

"AbilityUnitTargetType" : "UNIT_TARGET_HERO"
AbilityUnitTargetType 描述
UNIT_TARGET_HERO 英雄类型的敌人

AbilityUnitAiTargetFlag

AI获得该技能后，对目标的一些操作

"AbilityUnitAiTargetFlag" : [

    "UNIT_AI_TARGET_RESEARCH",

    "UNIT_AI_TARGET_RECORD"

]
AbilityUnitAiTargetFlag 描述
UNIT_AI_TARGET_RESEARCH 每次都会重新搜索目标，使用AbilityUnitAiTargetCondition规则
UNIT_AI_TARGET_RECORD 选取目标后，会记录该目标
UNIT_AI_TARGET_CLEAN 技能施放成功后(OnSpellStart)，会清空记录的目标。如果配置了该字段，又配置了UNIT_AI_TARGET_RECORD, UNIT_AI_TARGET_RECORD将会无效化

AbilityUnitAiTargetCondition

AI获得该技能后，选取目标的规则条件

"AbilityUnitAiTargetCondition" : "UNIT_TARGET_RANGE_MIN"
AbilityUnitAiTargetCondition 描述
UNIT_TARGET_RANDOM 随机一个目标
UNIT_TARGET_RANGE_MIN 距离最近的目标
UNIT_TARGET_RANGE_MAX 距离最远的目标
UNIT_TARGET_HP_MIN 当前血量最少的目标
UNIT_TARGET_HP_MAX 当前血量最多的目标
UNIT_TARGET_HP_PERCENT_MIN 当前血量与最大血量比值最小的目标
UNIT_TARGET_HP_PERCENT_MAX 当前血量与最大血量比值最大的目标
UNIT_TARGET_ENERGY_MIN 当前能量最少的目标
UNIT_TARGET_ENERGY_MAX 当前能量最大的目标
UNIT_TARGET_SLG SLG专用

AbilityCastRange

技能的施法距离。
0 表示施法者自己，会忽略AbilityUnitTargetTeam和AbilityUnitTargetType的配置。
-1 表示全屏目标，会根据AbilityUnitTargetTeam和AbilityUnitTargetType的配置来筛选

"AbilityCastRange" : -1

AbilityCastPoint

技能的施法前摇，如果技能在前摇内被打断，技能会释放失败，但是不进入CD

"AbilityCastPoint" : 0.3

AbilityCastAnimation

技能的施法动画

"AbilityCastAnimation" : "attack"

AbilityCastDuration

技能的持续时间

"AbilityCastDuration" : 0.3

AbilityChannelTime

技能持续施法的持续时间

"AbilityChannelTime" : 5

AbilityCooldown

技能的CD

"AbilityCooldown" : 20

AbilityInitialCooldown

技能初始CD，可不配置，默认是0

"AbilityInitialCooldown" : 20

AbilityCostType

释放技能所需要的消耗

AbilityCostType 描述
NO_COST 无消耗
ENERGY 能量消耗
"AbilityCostType" : "ENERGY"

AbilityCostValue

"AbilityCostValue" : 1000

AbilityAoeRadius

圆形AOE技能的AOE范围配置，与ABILITY_BEHAVIOR_RADIUS_AOE配合使用

"AbilityAoeRadius" : 5

AbilityAoeLine

线形AOE技能的AOE范围配置，与ABILITY_BEHAVIOR_LINE_AOE配合使用
Length 线的长度
Thickness 线的宽度

"AbilityAoeLine":

{

    "Length" : 5,

    "Thickness" : 3

}

扇形AOE技能的AOE范围配置，与ABILITY_BEHAVIOR_SECTOR_AOE配合使用
Radius 扇形半径范围
Angle 扇形的角度

"AbilityAoeSector":

{

    "Radiue" : 5,

    "Angle" : 30

}

预载Precache

用于资源的预加载。只有在这边配置的资源才会在战斗载入的时候进行预加载

"Precache":

[

    "effect/hero/heroFX07_02_attack"

]

技能事件AbilityEvents

技能事件在技能释放过程中根据触发条件触发。一个技能条件下可以配置任意个Action

Ability Event 触发条件
OnAbilityPhaseStart 技能开始施法。这个事件里面目前只能配置表现型的Action，不要配置逻辑相关的Action
OnAbilityPhaseCharge 技能开始蓄力。这个事件里面目前只能配置表现型的Action，不要配置逻辑相关的Action
OnSpellStart 当施法动画AbilityCastPoint完成后
OnChannelInterrupted 持续施法被中断
OnChannelFinish 持续施法成功或者中断，都会触发该事件
OnChannelSucceeded 经过持续施法时间AbilityChannelTime，完整持续施法
OnOwnerSpawned 拥有该技能的单位出生
OnOwnerDied 拥有该技能的单位死亡
OnProjectileHitUnit 子弹击中目标(通用回调，如果子弹自己定制了击中目标的回调，这个就不触发)

目标

在Action执行时，首先应该选中目标(Target)。所以基本每个Action都有Target属性。

单体目标

"Target" : "TARGET"
目标名称 描述
CASTER 施法者
TARGET 目标
POINT 点
ATTACKER 攻击者

多个目标

想要在某个区域里选中多个目标，需要指定一个中心点。
Center 以目标为搜索范围的中心
Teams 同AbilityUnitTargetTeam
Types 同AbilityUnitTargetType
AoeArea 目标搜索区域

搜索区域类型 描述
DAMAGE_AREA_TYPE_RADIUS 圆形搜索区域
DAMAGE_AREA_TYPE_LINE 线形搜索区域
DAMAGE_AREA_TYPE_SECTOR 扇形搜索区域

圆形区域配置

"AoeArea" :

{

    "AreaType" : "DAMAGE_AREA_TYPE_RADIUS",

    "Radius" : 5

}

线形区域配置

"AoeArea" :

{

    "AreaType" : "DAMAGE_AREA_TYPE_LINE",

    "Line" :

    {

        "Length" : 5,

        "Thickness" : 3

    }

}

扇形区域配置

"AoeArea" :

{

    "AreaType" : "DAMAGE_AREA_TYPE_SECTOR",

    "Sector" :

    {

        "Radius" : 5,

        "Angle" : 30

    }

}

Target示例

"Target" : 

 {

     "Center" : "CASTER",

     "Teams" : "UNIT_TARGET_TEAM_ENEMY",

     "Types" : "UNIT_TARGET_HERO",



     "AoeArea" :

     {

         "AreaType" : "DAMAGE_AREA_TYPE_LINE",

         "Line" :

         {

            "Length" : 5,

            "Thickness" : 3

         }

     }

 }

修改器Modifier

这个区域定义了一系列的Modifiers（一般可以认为是Buff或者DeBuff）。
目前Modifier通过名字来确定是不是同一个Modifier，也就是名字不一样的Modifier可以同时存在。如果想让一个Modifeir只能存在一个在目标身上，需要Modifier名字一样，并且Modifier属性里设置了不可叠加类型。

一些Modifier中的类型定义

Attributes 描述
MODIFIER_ATTRIBUTE_NONE 无，默认覆盖相同名字的Modifier
MODIFIER_ATTRIBUTE_MULTIPLE 可叠加Modifier

Modifier 参数

Modifier参数 描述
Attributes 不配置, 默认为MODIFIER_ATTRIBUTE_NONE
Duration 持续时间，不配置，默认为-1, 表示无限持续时间
ThinkInterval 循环间隔, 不配置，默认为-1, 只有>0才会触发OnIntervalThink事件
IsPassive 是否被动，为true将会在战斗开始的时候，自动生效。不配置，默认为false
IsBuff 是否是BUFF，不配置，默认为false
IsDebuff 是否是BUFF，不配置，默认为false
IsHidden 是否隐藏，不配置，默认为false
光环 需要光环就配置这些值
Aura 光环名字，拥有这个键值的Modifier将会把指定为Aura值的Modifier应用给Aura_Radius范围内的每个有效单位，并且在他们离开范围时移除
Aura_Radius 光环半径, -1表示全屏
Aura_Teams 同AbilityUnitTargetTeam
Aura_Types 同AbilityUnitTargetType
特效 需要特效就配置这些值，或者可以在OnCreated中使用AttachEffect代替
EffectName 特效名字
EffectAttachType 特效附着点
Properties Modifier会更改的属性列表
States Modifier会更改的状态列表

Modifier 事件

触发事件名称 描述
OnCreated 当Modifier被创建时
OnDestroy 当Modifier被销毁时
OnIntervalThink 循环执行指定的操作
OnDealDamage 当拥有modifier的单位施加伤害的时候
OnTakeDamage 当拥有modifier的单位受到伤害的时候
OnFriendlyDied 当友方有单位死亡时
OnEnemyDied 当敌方有单位死亡时
OnHeroDied 有单位死亡时

Modifier 属性

现在支持2种配置方法:

  • [10, 10]

  • ValueSource

"Properties" : 

{

    "MODIFIER_PROPERTY_PHYSICAL_ATTACK" : [10, 30],

    "MODIFIER_PROPERTY_PHYSICAL_DEFENCE" : [20, 30]

}



"Properties" : 

{

    "MODIFIER_PROPERTY_PHYSICAL_ATTACK" : 

    {

        "ValueSource" : 

        {

            "ValueBasicParams":[0, 0],

            "ValueAdditionParams":

            [

                {

                    "ValueSourceType":"SOURCE_TYPE_PHYSICAL_ATTACK",

                    "ValueSourceParams":[0, 0, 0, 0]

                }

            ]

        }

    }

}
属性名称 描述
MODIFIER_PROPERTY_PHYSICAL_ATTACK 修改物理攻击
MODIFIER_PROPERTY_PHYSICAL_DEFENCE 修改物理防御
MODIFIER_PROPERTY_PHYSICAL_PENETRATION 修改物理穿透
MODIFIER_PROPERTY_MAGIC_ATTACK 修改魔法攻击
MODIFIER_PROPERTY_MAGIC_DEFENCE 修改魔法防御
MODIFIER_PROPERTY_MAGIC_PENETRATION 修改魔法穿透
MODIFIER_PROPERTY_CRIT 修改暴击
MODIFIER_PROPERTY_DODGE 修改闪避数值
MODIFIER_PROPERTY_HIT_RATE 修改命中率
MODIFIER_PROPERTY_LIFELINK_LEVEL 修改吸血等级
MODIFIER_PROPERTY_MAX_HP 修改最大生命
MODIFIER_PROPERTY_STATS_STRENGTH_BONUS 修改力量
MODIFIER_PROPERTY_STATS_AGILITY_BONUS 修改敏捷
MODIFIER_PROPERTY_STATS_INTELLECT_BONUS 修改智力
MODIFIER_PROPERTY_INCOMING_TAKE_HEAL_VALUE_PERCENTAGE 修改受到的治疗效果万分比，负数降低治疗效果，正数增加治疗效果
MODIFIER_PROPERTY_INCOMING_DEAL_HEAL_VALUE_PERCENTAGE 修改造成的治疗效果万分比，负数降低治疗效果，正数增加治疗效果
MODIFIER_PROPERTY_INCOMING_TAKE_PHYSICAL_DAMAGE_PERCENTAGE 修改受到的物理伤害万分比，负数降低伤害，正数增加伤害
MODIFIER_PROPERTY_INCOMING_DEAL_PHYSICAL_DAMAGE_PERCENTAGE 修改造成的物理伤害万分比，负数降低伤害，正数增加伤害
MODIFIER_PROPERTY_INCOMING_TAKE_MAGIC_DAMAGE_PERCENTAGE 修改受到的魔法伤害万分比，负数降低伤害，正数增加伤害
MODIFIER_PROPERTY_INCOMING_DEAL_MAGIC_DAMAGE_PERCENTAGE 修改造成的魔法伤害万分比，负数降低伤害，正数增加伤害
MODIFIER_PROPERTY_INCOMING_TAKE_ALL_DAMAGE_PERCENTAGE 修改受到的所有伤害万分比，负数降低伤害，正数增加伤害
MODIFIER_PROPERTY_INCOMING_DEAL_ALL_DAMAGE_PERCENTAGE 修改造成的所有伤害万分比，负数降低伤害，正数增加伤害
MODIFIER_PROPERTY_PHYSICAL_SHIELD 物理护盾, 负数为吸收治疗量，正数为吸收伤害(护盾类型的Property不要与其他Property在同一个Modifier中一起使用。因为护盾被打破的时候，会清除它所在的Modifier。除非想配置一种拥有护盾的时候，攻击力提升或者其他属性改变的Modifier)
MODIFIER_PROPERTY_MAGIC_SHIELD 魔法护盾, 负数为吸收治疗量，正数为吸收伤害(同上)
MODIFIER_PROPERTY_SHIELD 物理魔法护盾, 负数为吸收治疗量，正数为吸收伤害(同上)
