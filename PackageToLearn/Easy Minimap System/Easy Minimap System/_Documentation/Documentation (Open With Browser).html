I am always willing to help, open to suggestions and always do my best to offer a quick and useful service to my customers! 😀 What is the Easy Minimap System? Easy Minimap System is a tool that allows you to create the Minimap of your game in minutes. The focus of the tool is speed and efficiency, you'll see how simple it is to have your minimap ready in few moments. Jump on this portal, if you want to go to the part where this documentation teaches you how to display GPS navigation routes using the Easy Minimap System! First steps to use Before you start creating a Minimap, it is recommended (but not mandatory) that you configure your game cameras, so that they do not capture the UI layer. You can do this by changing the Culling Mask variable on your cameras. Remembering that this is not mandatory, but it is highly recommended if you have cameras that have an extremely high far value (over 90000). How to create a Minimap Now, get to work. In this topic, we will learn how to create a basic Minimap using the Easy Minimap System. After reading this topic, you will be able to create your own Minimap. Everything will be divided into steps.

Step 1 - Add the "Minimap Camera" component to your player. This component renders all Minimap Items that are within your field of view. You can adjust the Capture Shape variable to define how the Minimap Camera will capture the surroundings. If you intend to create a square Minimap (with equal height and width), use the option Square. If you intend to create a rectangular or full screen minimap, use the Rectangular option. You can adjust the Capture Resolution variable until you reach the capture quality you want. Note that the higher the resolution, the more performance can be consumed. Step 2 - Add the "Minimap Item" component to all GameObjects you want to display on the minimap. You can add this component to your Player or Enemies for example. By changing the value of the Particle Highlight Mode variable, you can turn on an effect that highlights your Minimap Item within the Minimap. Change the Order In Layer variable to define whether this Minimap Item should be rendered above the others or not. The higher the value, the higher the Minimap Item will be rendered. Step 3 - Create an empty GameObject in your UI Canvas and add the "Minimap Renderer" component. This component is responsible for taking the image of a target Minimap Camera and displaying it in the UI Canvas. The first thing you should do after adding the "Minimap Renderer" to a GameObject from Canvas, is to drag a GameObject (containing a Minimap Camera) onto the Minimap Camera To Show variable. After that, Minimap Renderer will be able to display the entire image captured by that Minimap Camera while your game is running. You are able to change the format of your Minimap Renderer easily by clicking on the Square and Circle icons that are next to the Base Shape Sprite variable. You can add Minimap Items to the Minimap Items For Highlight list so that they are highlighted. If a Minimap Item is highlighted in the Minimap Renderer, whenever this Minimap Item is out of sight, it will be shown at the border of the Minimap. Step 4 - Now just test it! Your Minimap Camera should accompany your player and display the Minimap Items in your scene. Keep reading the documentation to learn how to optimally display your scenario on the Minimap, and to learn more!

Minimap Renderer: How to create a World Map (Fullscreen Map) To create a Fullscreen Map, everything is as simple as it looks!

Create a Minimap Renderer on your Canvas. Adjust it using anchors and etc (as you would adjust any other element of the Canvas) so that it covers your entire canvas. Adjusting it in this way, it will already cover your entire screen. Now, go to the Minimap Camera that captures things to display in your Minimap Renderer.

You need to set Capture Shape to Rectangular and Capture Resolution to a higher, rectangular resolution. Like 1280x720 or greater. You can choose a resolution according to the quality you want. You can also increase the Field of View if you want! So the Minimap Camera will capture an even larger area! In this way, the camera will capture things in a Rectangular format (in the form of screens) and will also capture things in good quality. Minimap Renderer: How to Zoom In/Out your Minimap By using the Easy Minimap System you can create a Zoom system for your Minimaps (this goes for Fullscreen Minimaps as well). In the Easy Minimap System demo scene you will find a demo of a Zoom control, more precisely in the FPS/TPS demo scene. Just press M to open the map in fullscreen and see how Zoom works.

First, you can zoom by manipulating the Field Of View of your Minimap Camera. Reducing the Field Of View of your Minimap Camera will have a Zoom In effect, but Increasing the Field Of View will have a Zoom Out effect. The Field Of View. When Reducing the Field Of View of your Minimap Camera, you will notice that the size of the Minimap Items will also increase. This is expected behavior as this may sometimes be desired. To reduce the size of your Minimap Items as Field Of View is reduced, you can control the Easy Minimap System Global Size Multiplier, this is quite simple, but you can have more details by clicking here.

Briefly, To Zoom In: Reduce the Field Of View of your Minimap Camera and reduce the Global Size Multiplier to reduce the size of your Minimap Items.

Briefly, To Zoom Out: Increase the Field Of View of your Minimap Camera and increase the Global Size Multiplier to increase the size of your Minimap Items. Minimap Renderer: How to receive Player Inputs (such as Touch/Clicks) and perform actions The Minimap Renderer has an API and Events kit so that you can receive data to manipulate the Inputs that a player makes (such as clicks or touches for example) on your Minimap Renderer area.

In the Minimap Renderer interface, you can register C# Methods from your scripts, so that these C# Methods are Called whenever the player Clicks, or Drag over the area of a Minimap Renderer for example. Thus, whenever a Player clicks on the Minimap Renderer area, for example, one of its methods will be called and some C# logic that you program will be executed. You can program, for example, whenever the player clicks on a place, that place will be marked.

Below you will see examples of codes to be implemented. The examples below use the Events API, and you can modify the codes below as needed. To allow your Minimap Renderer to receive the player's events, avoid leaving other elements of the canvas (such as Images for example) on top of your Minimap Renderer, because if there is any other element on top of your Minimap Renderer, it will intercept the touches or player inputs. You can also disable the Raycast Target options for UI elements that need to be on top of your Minimap Renderer, to prevent this from happening. Minimap Routes: Setting the height tolerance for Route calculations This topic tries to better explain how to set the startingHeightTolerance variable in the Minimap Routes component. But first, we need to understand how Minimap Routes calculates routes between the Start and Destination points.

When Minimap Routes calculates the route between the Start point and the Destination point, it follows a few steps... Obtains the current location of the Start point Gets the current height at which the Start point is Search for the Waypoint closest to the Start point. The Waypoint must also be at a height close to the height of the Starting point. This nearby height has a limit and that limit is defined by variable startingHeightTolerance. Disregards any Waypoints that have a height difference greater than the current height of the (Starting Point + startingHeightTolerance) After finding the Waypoint closest to the Starting point, using this rule, the route is calculated and then the best Waypoints are selected to compose the route between Starting and Destination. The route is rendered on the Minimap. If you still don't fully understand, the image below may help to clarify things further. Note the height of the Player, the height of the Bridge, the height of the Waypoints on the ground and the height of the Waypoints on the Bridge. The yellow mark represents the startingHeightTolerance variable. And this yellow representation moves along with the height of our Player, that is, if our Player was on the ground, the mark would also be on the ground.
If we increase the value of variable startingHeightTolerance, that yellow mark also increases.
Knowing this, any Waypoint that is outside the height of that yellow mark will be ignored in step 3 and will not be considered during the route calculation.

But why is this necessary?
Without this variable, Waypoints that are at a height very different from the height of the Player, would be considered and thus the routes would leave incorrect points, and from incorrect heights. In the image, for example, the Waypoints that are on the ground would be considered when calculating the start of the route and this is not correct, as the Player is over the bridge. With variable startingHeightTolerance , this can be avoided. Just adjust to a comfortable height and unnecessary Waypoints will be ignored during the calculation at the beginning of the route. Usually the recommended height is 3.5 but you can lower it if you need to, however, you will need to create more Waypoints in regions that are very steep. You can also use the onStartLoadState and onDoneLoadState events to know when a load has started or completed, so you have even more security to use the methods! Minimap Fog: How to use Minimap Fog? To get started, add the Minimap Fog component to a GameObject in your scene. It is recommended that this GameObject is empty and has no children GameObjects.

The Minimap Fog works on a Chunks principle, this means that you need to determine which area of your scene will be covered by the Chunks that will contain the Fogs while your game is running. As soon as you add the Minimap Fogs component to a GameObject, you will see a marker that will represent your first Chunk in your scene. Use the yellow arrows to increase and create new Chunks until your entire scene is covered. Now, you need to choose the method in which the Fog will be removed. You can do this by changing the Fog Removal Method. Finally, now you just need to show Minimap Fog which Transforms (targets) will be able to remove the Fogs around each one. Now, just start your game and test it! Minimap Fog: How to Save/Load the Current State of Fog The Minimap Fog component supports loading and saving of Fog States. When saving the current Fog State of your scene, Minimap Fog saves all already explored and unexplored locations to a file. This file is automatically managed internally by Minimap Fog. You can load this file (fog state) later whenever you want.

First you need to understand how Minimap Fog references each Fog State (or file) to know which one to load, or which one to save. In the Fogs Revealed Save ID variable you can define an ID that will refer to the Fog State you want Minimap Fog to save or load. For example. When you use Minimap Fog's C# API to save the current Fog State, Minimap Fog will use the ID that is currently entered in Fogs Revealed Save ID to save the Fog State. Later, when you want to load a Fog State, Minimap Fog will use the ID entered in Fogs Revealed Save ID to look for the Fog State that uses that ID. Incredible Tips That Can Be Useful: How to Edit Your PNG Texture Generated by Minimap Scanner When you scan your scene, using the Minimap Scanner, the component will save the PNG image generated from the scan, in the "Assets/MT Assets/_AssetsData/Scans" path. Knowing this, you can edit the generated PNG images, in an external Image Editor, or even use them as a basis to create stylized images maps. After finishing editing in an external Image Editor, just save the image and import it into your project files. After that, just drag your new edited PNG image into the Texture Result Of Scan variable of your Minimap Scanner component. If you edit the PNG image generated from the Minimap Scanner, you must maintain the original resolution of the generated PNG image. Do not resize the image to avoid possible problems with rendering and positioning your PNG image on the map. The image must have the same resolution selected in scanResolution on the Minimap Scanner.
